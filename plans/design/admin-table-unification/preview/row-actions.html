<RowActions>

At-rest, rows show only data. On hover the row tints, the per-row checkbox, edit pencil, and delete trash fade in (150ms ease-out), and the surface-special slot renders a labeled button when the surface needs one (e.g. Teams "Mark paid" only when payment status is pending). Photos is exempt from this rule per D7 — its moderation buttons stay always-visible.

Token spec — <RowActions>

TokenValueTailwind v4 classNotes
Icon button size 28 × 28px size-7 rounded-[var(--radius-md)] Matches existing icon-sm Button variant. ≥44pt minimum is met by the row hit-target as a whole, not the icon alone.
Icon glyph size 16 × 16px (lucide stroke 2) [&_svg]:size-4 Pencil + trash from lucide-react: Pencil, Trash2.
Per-row checkbox 16 × 16px h-4 w-4 (PR-1B fix) See checkbox preview for border + fill spec.
Inter-control spacing 4px gap-1 Tight cluster — controls read as a single group, not three buttons.
Surface-special left margin 4px (additional) ml-1 Visual separation from the icon cluster; signals it's a different action class.
Opacity transition 0 → 1 over 150ms ease-out opacity-0 group-hover:opacity-100 transition-opacity duration-150 ease-out Use a group on <tr> + group-hover on the actions container. Also reveal on focus-within for keyboard nav.
Row hover tint brand-muted / 45% hover:bg-brand-muted/45 Subtle — the row tint signals interactivity; the controls signal what's possible.
Selected row tint brand-muted / 65% data-[state=selected]:bg-brand-muted/65 When the per-row checkbox is checked, row tint persists + actions stay revealed.
z-index 1 (above row content) relative z-10 on the actions container Ensures icon hit targets sit above any cell-content overflow.
Trash hover treatment destructive / 10% bg hover:bg-destructive/10 hover:text-destructive Color paired with icon shape — destructive intent is never color-only.
Surface-special button Outline variant, sm size variant="outline" size="sm" Labeled button. Per surface, the slot may be empty (Contacts/Sponsors/Sponsorships) or present (Teams "Mark paid").
Hidden checkbox column behavior Reserved 40px column even at rest w-10 px-4 on <th> + first <td> Checkbox column is part of the table grid even when invisible — prevents column shift on hover.
D7 exemption (Photos) Approve / Reject / Delete always-visible n/a — Photos uses its own moderation component RowActions is NOT used on Photos. Moderation = primary task; hover-only would hide the work.
aria-label format "Edit {item}", "Delete {item}", "Select {item}" n/a — passed via props Aria-locked strings; F-N12 pattern. Header select-all uses aria-label="Select all".
